Гид по конфигу WebAR

Все параметры из css/config.css и js/config.js — с живыми примерами. Двигайте слайдеры и жмите кнопки, чтобы увидеть эффект НА САМОМ ДЕЛЕ (страница использует те же CSS/JS файлы, что и основное приложение). Эта страница — только для вас, в само приложение не встроена и не нужна на складе.

Ширина карточки --panel-width-vw / --panel-width-max

Итоговая ширина — МЕНЬШЕЕ из двух: доля экрана (vw) или потолок в пикселях. Тут меняем px-потолок.

Положение над меткой --panel-offset-y

Ближе к 0% — карточка ниже (ближе к метке). Дальше от 0% — выше.

Скругление и размытие --panel-radius / --panel-blur-px

Скругление углов карточки и сила блюра фона за ней ("матовое стекло"). 0 — совсем без блюра, быстрее на слабых устройствах.

Фото товара --item-thumb-size / --panel-collapsed-thumb-size / --item-thumb-bg

Размер фото отдельно для свёрнутой карточки (видно по умолчанию) и развёрнутой (после тапа). Фон под фото — белый по умолчанию: тёмный фон делал прозрачные поля PNG-фото чёрной рамкой (см. кнопку ниже).

Скорость анимаций --transition-position / --transition-expand / --transition-fade

position — слежение за меткой при движении камеры. expand — разворот по тапу и раздвигание при столкновении. fade — появление/исчезание. Жмите «Пульс», чтобы увидеть скорость в деле.

Сглаживание позиции POSITION_SMOOTHING

Красная точка — сырые координаты метки от детектора (могут дёргаться). Бирюзовая карточка — итоговая позиция. Жмите «Плохой кадр» — точка прыгает случайно в сторону. Смотрите, насколько сильно/слабо дёргается карточка.

сырая точка детектора карточка (сглаженная)

Разъезд карточек при столкновении COLLISION_STEP_PX / COLLISION_RESOLVE_INTERVAL_MS / COLLISION_PADDING_PX

STEP — на сколько px карточка сдвигается за один шаг. INTERVAL — как часто пересчитывать (больше = спокойнее, меньше = быстрее реагирует, но может дёргаться). PADDING — с какого расстояния карточки считаются "наехавшими". Если карточка А развёрнута — она НЕ двигается (см. предыдущий баг-фикс), карточка B её обходит.

Тайминги распознавания NO_DETECTION_TIMEOUT_MS / RESCAN_COOLDOWN_MS / PANEL_MAX_AGE_MS

Три независимых таймера — жмите кнопки по очереди.

Опрос камеры SCAN_INTERVAL_MS / ZXING_SCAN_INTERVAL_MS

Как часто пытаемся распознать штрихкод в кадре. Меньше — отзывчивее, но больше нагрузка на телефон. Точка "тикает" с этой частотой.

Как применить в бою: подобрали значение слайдером — открываете реальный css/config.css или js/config.js, находите переменную по имени (оно написано над каждым слайдером) и ставите то же число. Не забудьте бампнуть версию у CSS-файлов в index.html (?v=N), иначе браузер может закэшировать старое.
